================================================================================
TECHNICAL ARCHITECTURE
================================================================================

FRAMEWORK & TECH STACK
- React 18+ (hooks: useState, useEffect, useMemo)
- Next.js 14 (App Router)
- Client-side components ('use client' directive)
- Inline JavaScript styling (no CSS files, no Tailwind)
- No external UI libraries

STYLING APPROACH
- Color constants from /src/components/shared/theme.js
- Dark theme with accent colors
- Responsive grid and flexbox layouts
- Consistent monospace font for data (JetBrains Mono)
- Sans font for labels (Inter)
- Inline hover states and transitions

STATE MANAGEMENT
- Local component state (useState)
- Effect hooks for data fetching (useEffect)
- Memoized calculations (useMemo) for performance
- No Redux, Context, or external state management
- Real-time form submission without optimistic updates

DATA FLOW ARCHITECTURE
- Tabs fetch primary data on mount and when date range changes
- Tables display paginated/filtered results
- Row click triggers modal
- Modal fetches additional detail data
- Form submission via PUT/POST to API endpoints
- Local state updates after successful API response

COMPONENT REUSABILITY
- KPICard (simple metric tile) - defined locally in each tab
- SortableTable (clickable headers) - defined locally in modals and tabs
- Section (card container) - defined locally in each component
- GoalTile (compact progress tile) - defined locally in BusinessHealthTab
- No shared component definitions to avoid circular dependencies

PERFORMANCE OPTIMIZATIONS
- useMemo for filtered and sorted data
- Lazy loading of modals (only when selected)
- useEffect dependencies properly scoped
- Debounced table sorting
- No unnecessary re-renders (explicit dependencies)
- Date calculations memoized

ACCESSIBILITY
- Semantic HTML structure (table, th, td, button)
- Color contrast meets WCAG standards
- Text alternatives for icons (badges, indicators)
- Keyboard navigable (tab, enter for forms)
- Focus states maintained

API INTEGRATION REQUIREMENTS
================================================================================

ENDPOINT: GET /api/crm/leads
Query Params: start, end, page, limit
Response: { leads: [{ leadId, firstName, lastName, phone, source, status, agent, lastContact, followUpDue, attempts, createdAt, notes }] }

ENDPOINT: GET /api/crm/lead/:leadId
Response: { leadId, firstName, lastName, phone, source, status, agent, lastContact, followUpDue, attempts, createdAt, notes, convertedPolicies }

ENDPOINT: GET /api/crm/lead/:leadId/calls
Response: { calls: [{ date, duration, campaign, status, type }] }

ENDPOINT: PUT /api/crm/lead/:leadId
Body: { status?, notes? }
Response: updated lead object

ENDPOINT: GET /api/crm/policyholders
Query Params: start, end, page, limit
Response: { policyholders: [{ policyNumber, firstName, lastName, carrier, product, premium, status, issueDate, agent, lastPaymentDate, lastOutreach, outreachAttempts }] }

ENDPOINT: GET /api/crm/policyholder/:policyNumber
Response: { policyNumber, firstName, lastName, carrier, product, premium, status, issueDate, agent, lastPaymentDate, lastOutreach }

ENDPOINT: PUT /api/crm/policyholder/:policyNumber
Body: { status?, lapseReason? }
Response: updated policyholder object

ENDPOINT: POST /api/crm/tasks
Body: { policyNumber, type, method, outcome, notes }
Response: { createdAt, method, outcome, agent, notes }

ENDPOINT: GET /api/crm/tasks
Query Params: policyNumber
Response: { tasks: [{ createdAt, method, outcome, agent, notes }] }

ENDPOINT: GET /api/crm/metrics/business-health
Query Params: start, end
Response: {
  totalActiveMembers, totalPremiumInForce, atRiskMembers, lapsedThisMonth, winBackSuccesses,
  monthlyLapseRate, winBackRate, persistencyRate, avgPolicyLifespan, revenueAtRisk,
  topDefectingCarrier, topLapseReason, highestLapseAgent, highestLapseRate,
  bestRetentionAgent, bestRetentionRate, outreachDueToday,
  lapseReasonBreakdown: [{ reason, count }],
  carriers: [{ carrier, activeMembers, lapsedMembers, lapseRate, premiumInForce, revenueAtRisk, persistencyRate }],
  timeSeries: [{ date, activeMembers, lapseRate, revenueAtRisk, winBackRate }]
}
