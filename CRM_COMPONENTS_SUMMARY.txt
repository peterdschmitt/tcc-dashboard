================================================================================
API DEPENDENCIES
================================================================================

Each component requires corresponding API endpoints:

LeadCRMTab:
  GET /api/crm/leads?start=YYYY-MM-DD&end=YYYY-MM-DD&page=1&limit=500
  Returns: { leads: [...] }

RetentionDashboardTab:
  GET /api/crm/policyholders?start=YYYY-MM-DD&end=YYYY-MM-DD&page=1&limit=500
  Returns: { policyholders: [...] }

BusinessHealthTab:
  GET /api/crm/metrics/business-health?start=YYYY-MM-DD&end=YYYY-MM-DD
  Returns: { totalActiveMembers, lapseRate, carriers: [...], timeSeries: [...], etc. }

LeadDetailModal:
  GET /api/crm/lead/:leadId
  Returns: { leadId, firstName, lastName, status, notes, convertedPolicies: [...] }
  
  GET /api/crm/lead/:leadId/calls
  Returns: { calls: [...] }
  
  PUT /api/crm/lead/:leadId
  Body: { status, notes }
  Returns: updated lead object

PolicyholderDetailModal:
  GET /api/crm/policyholder/:policyNumber
  Returns: { policyNumber, firstName, lastName, status, premium, etc. }
  
  GET /api/crm/tasks?policyNumber=XXXX
  Returns: { tasks: [...] }
  
  PUT /api/crm/policyholder/:policyNumber
  Body: { status, lapseReason }
  Returns: updated policyholder object
  
  POST /api/crm/tasks
  Body: { policyNumber, type, method, outcome, notes }
  Returns: created task object

================================================================================
USAGE IN MAIN DASHBOARD
================================================================================

Import these tabs in Dashboard.jsx:

import LeadCRMTab from './tabs/LeadCRMTab';
import RetentionDashboardTab from './tabs/RetentionDashboardTab';
import BusinessHealthTab from './tabs/BusinessHealthTab';

Add to TABS array:
{ id: 'leads', label: 'Leads CRM' },
{ id: 'retention', label: 'Retention Dashboard' },
{ id: 'business-health', label: 'Business Health' },

Add to tab switch:
case 'leads':
  return <LeadCRMTab dateRange={dateRange} />;
case 'retention':
  return <RetentionDashboardTab dateRange={dateRange} />;
case 'business-health':
  return <BusinessHealthTab dateRange={dateRange} />;

================================================================================
KEY FEATURES & ARCHITECTURE
================================================================================

1. Component Composition:
   - Tabs import corresponding Modal components
   - Modals use shared SortableTable component
   - All styling inline JavaScript (no external CSS)
   - No circular dependencies

2. State Management:
   - useState for data fetching, loading, error, UI state
   - useEffect for API calls triggered by dateRange changes
   - useMemo for filtered/sorted data calculations
   - Real-time metric calculations (conversion rates, days since, etc.)

3. Data Flow:
   - Tab component fetches primary data → stores in state
   - Row click triggers detail modal → modal fetches additional data
   - Modal forms POST/PUT to APIs → updates local state

4. Reusable Patterns:
   - KPICard component: displays metric with optional goal
   - SortableTable component: click headers to sort, row click handlers
   - Section wrapper: card container with title header
   - GoalTile component: compact tiles with progress bars (BusinessHealthTab)

5. Date Formatting:
   - Consistent toLocaleDateString() for all dates
   - Color-coded relative dates (overdue red, future green)
   - Days calculation: Math.floor((new Date() - new Date(val)) / (1000*60*60*24))
